安装ipython出现的问题

ipython

在mac电脑上安装ipython出现了一些问题,折腾了很长时间,记录一下.
首先用 pip install ipython 安装ipython

Requirement already satisfied: ipython in /Library/Python/2.7/site-packages

Requirement already satisfied: prompt-toolkit<2.0.0,>=1.0.4 in /Library/Python/2.7/site-packages (from ipython)

Requirement already satisfied: decorator in ./Library/Python/2.7/lib/python/site-packages (from ipython)

Requirement already satisfied: setuptools>=18.5 in ./Library/Python/2.7/lib/python/site-packages (from ipython)

Requirement already satisfied: pickleshare in /Library/Python/2.7/site-packages (from ipython)

但是在你用ipython时,却出现

~ ipython
zsh: command not found: ipython

原因是因为 Mac OS X 10.11（El Capitan）默认开启了 SIP（System Integrity Protection）功能，一些系统文件就算使用「sudo」大法也无法搞定，比如安装 IPython，安装 XtraFinder.app，在 SIP “保护”下都不能完美安装使用。

解决办法如下:
首先重启系统，在开机声音响起的同时按住 Command + R 键，当出现苹果 Logo 黑白画面的时候松开键盘，进入恢复模式的操作面板，然后打开「终端」，输入：csrutil disable
执行命令后再重启系统，这样 SIP 功能就关闭了。进入系统后打开「终端」，前提是已经安装了 pip，然后输入：pip install ipython
这样一般问题就解决了

ipython3

但是我在安装ipython3 pip3 install ipython时,又出现了上述问题.
解决办法:

  • 我使用

    virtualenv venv --python=python3.5

    创建了一个虚拟环境

  • 使用命令进入虚拟环境

    source venv/bin/activate

  • 再安装ipython3

    pip3 install ipython

  • 然后在venv/bin 里即有 ipython3 文件

  • 将其复制到 /usr/local/bin 目录下

    sudo cp venv/bin/ipython3 /usr/local/bin/

这样就可以用ipython3了.

还有一种解决办法:

确保 /Library/Frameworks/Python.framework/Versions/3.6/bin 在你的python路径PATH下

添加路径到:(3.6是你的python版本)

export PATH=/Library/Frameworks/Python.framework/Versions/3.6/bin:$PATH

这个亲测可用!但是关闭终端,再使用IPython3就不能用了,必须重新输入上面的命令才能使用,是一个治标不治本的办法.

以上就是我在mac下安装ipython碰见的问题和解决办法.
